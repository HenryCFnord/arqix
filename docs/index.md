# arqix docs

This `docs/` tree is the first corpus for the project itself.

It is meant to work in three modes at once:

- as normal GitHub-readable documentation
- as a future input corpus for the `arqix` CLI
- as a portable Markdown knowledge base that can also be opened in tools such as Obsidian

The structure is intentionally simple.

## Start Here

- [Project documents](project/index.md)
- [Blog](blog/2026-03-20-why-arqix-had-to-exist.md)

- [Experiments](experiments/README.md)

## Directory Guide

- `project/` contains relatively stable project-level documents
- `blog/` contains reflective but technically grounded writing
- `notes/` contains informal working notes worth keeping
- `experiments/` contains explicit experiments with a clear scope

## Working Assumption

If `arqix` becomes useful, it should happen by improving ordinary documentation practice rather than by replacing it with a closed system. That is why the repository starts with plain Markdown and ordinary links.
