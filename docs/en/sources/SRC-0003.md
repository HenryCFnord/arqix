## EARS Requirement Syntax Patterns

The EARS pattern set (Easy Approach to Requirements Syntax, Mavin et al., RE'09), captured as the second normative base of this repository's requirement language.
The requirements style guide `docs/en/processes/requirements-style-guide.md` derives its sentence patterns from EARS — ubiquitous, event-driven, state-driven, unwanted behaviour, optional feature, complex — and the requirements checker classifies every normative sentence against them (REQ-01-01-11-06).
