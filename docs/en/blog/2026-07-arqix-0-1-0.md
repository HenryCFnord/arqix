# arqix 0.1.0: the tool that verifies its own announcement

arqix 0.1.0 is out.
It is on [crates.io](https://crates.io/crates/arqix), so the whole install story is:

```bash
cargo install arqix
```

## What shipped

The short version: a Rust CLI that treats a Markdown corpus as a typed graph and verifies it.

- A document parser, store, and catalog: frontmatter as data, relations as triples, everything queryable as JSON.
- A linter that checks the contracts prose cannot keep by itself — duplicate ids, unresolved includes and references, lifecycle vocabularies, and the machine-checked done claim: a story may only claim `done` when every requirement it carries is verified by an active test.
- A formatter built on a lossless line CST, byte-identical and idempotent on this repository.
- A trace engine that connects requirements to the `arqix:verifies` and `arqix:implements` markers in code, with coverage by requirement kind — and a ratchet that gates coverage *regressions* while leaving growth free.
- A verification orchestrator (`arqix verify`) that runs all of it as one configured policy.
- A publisher that stages artefact-ready pages and orchestrates a site toolchain — arqix never renders; [Zensical](https://zensical.org) does, exactly as Pandoc will for PDF.

The full list is in the [changelog](https://github.com/HenryCFnord/arqix/blob/main/CHANGELOG.md).

## The part we care about

This release was not announced first and verified later.
The repository gate that guards every change now runs `arqix verify` itself — the product checks the corpus that specifies the product.
[arqix.dev](https://arqix.dev) is staged and published by `arqix publish site` from the same corpus.
And the release you are reading about was validated by a pipeline that re-runs the whole gate against the exact tagged state before a human clicks publish.

The numbers stay honest in the same way.
The specification is deliberately ahead of the implementation: the [scoreboard](../reports/units/scoreboard.md) on the landing page is a generated report unit, included by an `arqix:include` directive and refreshed with every build.
Nobody edits that number by hand, which is precisely why it can be trusted when it is unflattering.

## What 0.1.0 is not

It is not 1.0.0.
Before that: the stitching mechanics for assembled documents (level-parameterised includes, heading-ownership policy — ADR-0013), the configured ID policy, report bundles, `render pdf`, and an MCP server so coding agents can use the graph directly.
The [roadmap](../project/roadmap.md) has the order; the scoreboard has the truth.

If your documentation should be part of your engineering system instead of sitting next to it — try the [Quick Start](../getting-started.md).
It runs in five minutes, and every output block in it is real.
